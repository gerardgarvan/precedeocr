    **2. `run_single_pdf_at_dpi(pdf_path, dpi)`**
    - This function processes a single PDF at a specific DPI setting
    - Must temporarily modify how process_single_pdf renders pages
    - Approach: Instead of modifying process_single_pdf directly, create a small rendering function that opens the PDF with fitz, renders each page at the given DPI, and runs extract_id_with_rotation on each page image
    - This avoids monkey-patching process_single_pdf
    - Implementation:
      ```python
      import fitz
      from PIL import Image
      from precede_ocr import extract_id_with_rotation

      def run_single_pdf_at_dpi(pdf_path, dpi):
          """Process a single PDF at a specific DPI, returning results list."""
          filename = Path(pdf_path).name
          try:
              doc = fitz.open(str(pdf_path))
          except Exception as e:
              return [{'filename': filename, 'page': 0, 'ids': [],
                       'rotation_detected': None, 'notes': f'error: {type(e).__name__}: {e}'}]
          try:
              results = []
              for page_idx in range(len(doc)):
                  page = doc[page_idx]
                  pix = page.get_pixmap(dpi=dpi, alpha=False)
                  img = Image.frombytes("RGB", [pix.width, pix.height], pix.samples)
                  ids_found, rotation, notes = extract_id_with_rotation(img)
                  results.append({
                      'filename': filename,
                      'page': page_idx + 1,
                      'ids': ids_found,
                      'rotation_detected': rotation if ids_found else None,
                      'notes': notes
                  })
              return results
          finally:
              doc.close()
      ```

    **3. `benchmark_dpi(pdf_paths, dpi_values=[200, 250, 300])`**
    - Per D-03: Test 200, 250, and 300 DPI on the sample
    - For each DPI value:
      - Record start time with `time.perf_counter()`
      - Process all sample PDFs sequentially using `run_single_pdf_at_dpi(pdf, dpi)` for each
      - Record elapsed wall-clock time
      - Count total pages processed, total IDs found
      - Calculate throughput: pages_per_second = total_pages / elapsed
    - Return pandas DataFrame with columns: `dpi, total_time_sec, pages_processed, ids_found, pages_per_second`
    - Print formatted comparison table after all DPI values tested

    - Return pandas DataFrame with columns: `workers, total_time_sec, pdfs_processed, pdfs_per_second`
    - Print formatted comparison table

    **5. `benchmark_whitelist(pdf_paths, sample_count=20)`**
    - Per D-05: Compare speed with vs without whitelist
    - Test on a smaller subset (20 PDFs) since this is a per-page OCR config test
    - Run each PDF twice: once with current config `'--psm 6 --oem 3 -c tessedit_char_whitelist=0123456789'` and once with `'--psm 6 --oem 3'` (no whitelist)
    - To swap configs, use `unittest.mock.patch` on the config string in precede_ocr module, OR directly call extract_id_with_rotation with temporarily patched pytesseract.image_to_string
    - Actually, simpler approach: render each page, run pytesseract.image_to_string directly with each config, time each call
    - Return DataFrame with columns: `config, total_time_sec, pages_processed, time_per_page_ms`
    - Print comparison showing percentage speedup (if any)

    **6. `validate_accuracy(sample_results, baseline_csv_path)`**
    - Per D-08: Page-by-page ID comparison against v1.1 baseline
    - Load baseline CSV (from v1.1 output) using pandas
    - Baseline CSV has columns: filename, folder_path, page, id, rotation_detected, notes
    - Group baseline by (filename, page) -> set of IDs
    - Group sample_results by (filename, page) -> set of IDs
    - For pages present in BOTH:
      - Match: baseline IDs == sample IDs
      - Mismatch: different IDs found
    - Calculate accuracy = matches / total_compared * 100
    - Print page-by-page comparison for mismatches (show filename, page, baseline IDs vs new IDs)
    - Return accuracy percentage (float)
    - Assert accuracy >= 94.0 and print PASS/FAIL

    **7. `main()` — CLI entry point:**
    - Use argparse with:
      - `corpus_dir` (positional): Path to PDF corpus directory
      - `--baseline-csv` (optional): Path to v1.1 baseline CSV for accuracy comparison
