# Architecture Patterns: v1.3 Results Cleanup & ID Lookup Integration

**Domain:** Post-Processing, Lookup Generation, and Error Investigation
**Researched:** 2026-06-09
**Confidence:** HIGH

## Executive Summary

This document maps how v1.3 results cleanup and ID lookup features integrate with the existing Precede OCR pipeline architecture. The current architecture produces `results.csv` (scan output) and `checkpoint.json` (per-file results) from the main OCR pipeline. V1.3 adds **post-processing capabilities** to transform scan output into lookup output and **error investigation tooling** to analyze pipeline failures. The recommended architecture uses **integrated CLI subcommands** rather than separate scripts, keeping all functionality within `precede_ocr.py` while maintaining clear separation of concerns through argparse subparsers.

## Current Architecture Overview (v1.0-v1.2)

### Component Structure

```
precede_ocr.py (main entry point)
├── Campaign Manager (state, resume menu, reporting)
├── Worker Pool (multiprocessing.Pool)
│   └── Worker Process (per PDF)
│       ├── PDF → Pages (PyMuPDF)
│       ├── Per-Page Processing
│       │   ├── Multi-Rotation OCR (pytesseract)
│       │   └── Preprocessing Fallback (OpenCV)
│       └── Result Accumulation
├── Checkpoint Writer (atomic .checkpoint.json)
├── Campaign State Writer (atomic campaign_state.json)
└── Output Generator (CSV + JSON)
    ├── results.csv (scan output: filename, page, id, rotation, notes)
    └── results.json (hierarchical: {filename: {page: [ids]}})
```

### Data Artifacts

| Artifact | Purpose | Schema | Generated By |
|----------|---------|--------|--------------|
| **results.csv** | Scan output (one row per ID) | `filename, page, id, rotation_detected, notes` | `write_results_csv()` |
| **results.json** | Hierarchical scan output | `{filename: {page: [ids]}}` | `write_results_json()` |
| **checkpoint.json** | Resume state (list of result dicts) | `[{filename, page, ids[], rotation_detected, notes, folder_path}]` | `process_all_pdfs()` |
| **campaign_state.json** | Campaign metadata | `{campaign_id, status, folder_stats, timestamps}` | `CampaignState.save()` |
| **batch_stats.json** | Summary metrics | `{total_files, successful, failed, total_pages, ids_found}` | `write_batch_stats()` |
| **campaign_report.md** | Human-readable summary | Markdown table with folder stats and problem highlighting | `generate_campaign_report()` |

### Current Data Flow

1. **Campaign Start**: Load or create `campaign_state.json`
2. **File Discovery**: Recursively glob `**/*.pdf`
3. **Dispatch**: `Pool.map(process_pdf, pdf_paths)`
4. **Worker Execution**: Each worker processes one PDF end-to-end
5. **Per-Page Loop**: Convert page → OCR with rotations → extract IDs → accumulate results
6. **Checkpoint**: Atomic write per-file results to `checkpoint.json`
7. **Aggregation**: Main process collects results, updates `campaign_state.json`
8. **Output**: Generate `results.csv` + `results.json` from accumulated checkpoint data

### Key Architectural Constraints

- **Windows spawn model**: Workers cannot share memory; must pass filenames not objects
- **Coarse-grained parallelism**: One PDF per worker (not page-level parallelism)
- **Atomic checkpoints**: Worker results must be JSON-serializable dicts
- **No IPC overhead**: Workers accumulate results locally, return to main process
- **Existing test coverage**: 236 tests verify pipeline behavior

## v1.3 Integration Points: Post-Processing & Error Investigation

### Problem Context

**Current state**: Production run complete with 30,365 files processed, producing:
- **results.csv**: Scan output with 52,055 IDs (one row per ID extracted)
- **49 failed files**: 46 FileNotFoundError, 3 EmptyFileError
- **59 no-match pages**: Pages where OCR found no ID
- **5,141 multi-ID pages**: Pages with multiple IDs (real duplicates vs OCR noise)

**v1.3 Goals**:
1. **Lookup generation**: Transform `results.csv` into ID lookup CSV sorted by ID (columns: ID, Filename, Page, Folder)
2. **Error investigation**: Analyze failed files and no-match pages to identify root causes
3. **Multi-ID cleanup**: Investigate multi-ID pages to distinguish real IDs from OCR noise
4. **Quality reporting**: Document findings and remaining issues

### Architecture Decision: Integrated Subcommands vs Separate Scripts

**Option A: Separate Utility Scripts** (NOT RECOMMENDED)
```
precede_ocr.py          # Main OCR pipeline
generate_lookup.py      # Post-processing script
investigate_errors.py   # Error analysis script
cleanup_multi_ids.py    # Multi-ID deduplication script
```

**Cons:**
- **Fragmented codebase**: 4 separate entry points, harder to maintain
- **Duplicated imports**: Each script imports pandas, pathlib, json separately
- **No shared utilities**: Common code (read checkpoint, parse results) duplicated
- **User confusion**: Which script to run when? No unified help/docs
- **Testing burden**: Separate test fixtures for each script

**Option B: Integrated CLI Subcommands** (RECOMMENDED)
```
python precede_ocr.py scan <dir>           # Main OCR pipeline (current)
python precede_ocr.py lookup <scan.csv>    # Generate lookup CSV from scan CSV
python precede_ocr.py investigate          # Error investigation (reads checkpoint)
python precede_ocr.py clean-multi-ids      # Multi-ID cleanup analysis
```

**Pros:**
- **Single entry point**: All functionality accessible via `precede_ocr.py`
- **Shared utilities**: Common functions (read checkpoint, parse CSV) in one module
- **Unified CLI**: `python precede_ocr.py --help` shows all available commands
- **Better testing**: Shared test utilities, consistent patterns
- **Cleaner dependencies**: One `requirements.txt`, one import strategy
- **Industry standard**: Git, Docker, AWS CLI all use subcommand pattern

**Evidence from Research:**
- [Command-Line Subcommands with Python's argparse](https://medium.com/@selvakumar-arumugapandian/command-line-subcommands-with-pythons-argparse-4dbac80f7110): "Many programs split up their functionality into subcommands (like svn checkout, svn update, svn commit)"
- [How to Build CLI Applications with argparse](https://oneuptime.com/blog/post/2026-02-03-python-argparse-cli/): "Keep subcommands simple and focused on a single task. Use consistent naming conventions across all subcommands."
- [Python CLI Tools with Click and Typer](https://devtoolbox.dedyn.io/blog/python-click-typer-cli-guide): "Subcommands allow programs to perform different functions requiring different command-line arguments"

**Recommendation:** **Option B (Integrated Subcommands)** — maintains code cohesion, follows industry best practices, reduces maintenance burden.

## Data Flow: Post-Processing Pipeline

### Flow 1: Lookup Generation

```
INPUT: results.csv (scan output)
  ↓
cmd_generate_lookup(args)
  ↓
read_scan_csv() → pandas DataFrame
  ↓
Filter: Remove rows where page == 0 (error records)
  ↓
Filter: Remove multi-ID noise (if --deduplicate flag set)
  ↓
Sort: df.sort_values(['id', 'filename', 'page'])
  ↓
Transform: df[['id', 'filename', 'page', 'folder_path']]
  ↓
Write: df.to_csv(output_path, index=False)
  ↓
OUTPUT: lookup.csv (sorted by ID)
```

**Lookup CSV Schema:**
```csv
id,filename,page,folder_path
10001,file1.pdf,1,folder_a
10002,file1.pdf,2,folder_a
10002,file2.pdf,5,folder_b  # Duplicate ID across files (expected)
```

**Key Transformations:**
1. **Filter error records**: `page == 0` indicates processing error (current convention)
2. **Sort by ID**: Primary sort key for Excel lookup (`VLOOKUP`, `INDEX/MATCH`)
3. **Deduplicate multi-ID noise** (optional): Remove OCR false positives based on heuristics
4. **Add folder_path**: Extract from `checkpoint.json` or parse from filename

**Pandas Operations:**
```python
import pandas as pd

def generate_lookup_csv(scan_csv_path, output_path, deduplicate=False):
    # Read scan CSV
    df = pd.read_csv(scan_csv_path)

    # Filter error records (page == 0)
    df = df[df['page'] > 0]

    # Optional: Deduplicate multi-ID noise
    if deduplicate:
        df = filter_multi_id_noise(df)

    # Sort by ID, then filename, then page
    df = df.sort_values(['id', 'filename', 'page'])

    # Select and reorder columns
    lookup_df = df[['id', 'filename', 'page']]

    # Add folder_path if available (from checkpoint.json or parse filename)
    # lookup_df['folder_path'] = extract_folder_paths(df)

    # Write lookup CSV
    lookup_df.to_csv(output_path, index=False)
    print(f"Lookup CSV written to {output_path} ({len(lookup_df)} rows)")
```

def investigate_failed_files(checkpoint_data):
    # Convert to DataFrame
    df = pd.DataFrame(checkpoint_data)

    # Filter error records (page == 0)
    errors = df[df['page'] == 0]

    # Extract error type from notes field
    errors['error_type'] = errors['notes'].str.extract(r'error: (\w+)')

    # Group by error type
    error_summary = errors.groupby('error_type')['filename'].apply(list).to_dict()

    # Filter no-match pages (page > 0, notes contains "no_match")

    # Categorize by preprocessing attempts

    # Group by category

### Flow 3: Multi-ID Cleanup

```
INPUT: results.csv (scan output)
  ↓
cmd_clean_multi_ids(args)
  ↓
read_scan_csv() → pandas DataFrame
  ↓
analyze_multi_id_pages()
  ├── Group by (filename, page)
  ├── Filter: Groups with len(ids) > 1
  ├── Analyze patterns:
  │   ├── Sequential IDs (likely real: 16384, 16385)
  │   ├── Sequence outliers (likely noise: 16384, 82885)
  │   ├── Confidence scores (from notes field: seq_outlier_conf_0%)
  │   └── Rotation consistency (same rotation = likely real)
  └── Return: {page_ref: [real_ids, noise_ids]}
  ↓
filter_multi_id_noise()
  ├── Apply heuristics: Keep IDs with high confidence, drop outliers
  ├── Mark ambiguous cases for manual review
  └── Return: cleaned DataFrame
  ↓
Write: cleaned_results.csv
  ↓
OUTPUT: cleaned_results.csv (deduped scan output)
```

**Multi-ID Deduplication Heuristics:**

1. **Sequence Validation** (existing in pipeline):
   - Uses Theil-Sen robust regression to detect sequence outliers
   - Already tracked in `notes` field: `seq_outlier_conf_X%`
   - **Heuristic**: Drop IDs with `seq_outlier_conf_0%` on multi-ID pages

2. **Rotation Consistency**:
   - If all IDs on a page have same rotation → likely real duplicates
   - If IDs have different rotations → likely OCR noise (multiple passes detected different text)
   - **Heuristic**: Keep IDs with consistent rotation, flag mixed rotations for review

3. **ID Range Clustering**:
   - Corpus IDs likely in contiguous ranges (e.g., 10000-20000)
   - IDs far outside observed range → likely OCR misreads
   - **Heuristic**: Flag IDs > 2 std deviations from mean for manual review

**Pandas Deduplication:**
```python
import pandas as pd

def filter_multi_id_noise(df):
    # Group by (filename, page)
    grouped = df.groupby(['filename', 'page'])

    def filter_group(group):
        if len(group) == 1:
            return group  # Single ID, keep as-is

        # Multi-ID page: apply heuristics

        # Heuristic 1: Remove seq_outlier_conf_0% on multi-ID pages
        group['is_outlier'] = group['notes'].str.contains('seq_outlier_conf_0%', na=False)
        if group['is_outlier'].any() and len(group) > 1:
            group = group[~group['is_outlier']]

        # Heuristic 2: Check rotation consistency
        rotations = group['rotation_detected'].nunique()
        if rotations > 1:
            # Mixed rotations: keep most common rotation only
            most_common_rotation = group['rotation_detected'].mode()[0]
            group = group[group['rotation_detected'] == most_common_rotation]

        # Heuristic 3: Range validation (if ID distribution known)
        # median_id = df['id'].median()
        # std_id = df['id'].std()
        # group = group[abs(group['id'] - median_id) < 2 * std_id]

        return group

    # Apply filter to each group
    cleaned = grouped.apply(filter_group).reset_index(drop=True)

    print(f"Filtered {len(df) - len(cleaned)} noise IDs from {len(df)} total")
    return cleaned
```

- **Code duplication**: Each script duplicates checkpoint reading, CSV parsing
- **User confusion**: Which script to run? No unified docs
- **Maintenance burden**: Changes to checkpoint schema affect 4 files
- **Fragmented testing**: Separate test fixtures for each script

**Verdict:** **Rejected** — integration benefits outweigh separation benefits for this use case

### Alternative 2: Integrated Functions (No CLI)

**Structure:**
```python
# precede_ocr.py
def main():
    # Run OCR pipeline
    results = process_all_pdfs(...)

    # Auto-generate lookup CSV
    generate_lookup_csv(results, 'lookup.csv')

    # Auto-investigate errors

```

**Pros:**
- Zero user intervention (fully automated)
- Simple workflow (run once, get all outputs)

**Cons:**
- **No user control**: Cannot skip lookup generation if not needed
- **Longer runtime**: Always runs post-processing even if user only wants scan
- **Harder to debug**: Cannot re-run post-processing without re-running OCR
- **Less flexible**: Cannot generate lookup from old scan CSV

**Verdict:** **Rejected** — subcommands provide better user control and flexibility

```python
# Pandas operations on 52K rows
df = pd.read_csv('results.csv')          # I/O: ~1s for 52K rows

df.to_csv('lookup.csv')                  # I/O: ~1s

```

**Error Investigation:**
```python
# Analysis on checkpoint.json (~30K files)
with open('checkpoint.json') as f:
    data = json.load(f)                  # I/O: ~2s for 30K records
df = pd.DataFrame(data)                  # Convert: <0.5s
errors = df[df['page'] == 0]             # Filter: <0.1s
summary = errors.groupby('error_type')   # Aggregate: <0.1s

```
