Present the following 5 manual verification tests to the user. These tests verify that the graceful shutdown infrastructure implemented in Plan 01 works correctly with real Ctrl+C signals on Windows 10. The executor does NOT modify any code — this is a human verification checkpoint.

**Setup:** The user needs a directory with at least 10-20 PDF files for testing. A small subset of their actual PDF collection works.

**Test 1: Graceful shutdown (SHUT-01, SHUT-02, SHUT-04, SHUT-05)**
1. Open a terminal (Command Prompt or PowerShell)
2. Run: `python precede_ocr.py YOUR_PDF_DIR --output-csv output/test_shutdown.csv --fresh`
3. Wait for the tqdm progress bar to appear and show a few files processed
4. Press Ctrl+C ONCE
5. Expected:
   - Message appears: "Ctrl+C received. Finishing in-flight files... (press Ctrl+C again to force-quit)"
   - Progress bar stops advancing
   - After a few seconds, summary appears: "Interrupted: X/Y files processed (Z IDs found). State saved. Resume with same command."
   - Terminal prompt returns cleanly (no garbled ANSI codes, cursor in correct position)
   - No Python error tracebacks visible

**Test 2: Campaign state verification (SHUT-05)**
1. After Test 1, open `output/campaign_state.json`
2. Verify: `"status": "interrupted"`
3. Verify: `"interruptions"` list has at least one entry with `"timestamp"` and `"reason": "user_interrupt"`

**Test 3: Resume after interrupt**
1. Run the same command again (without --fresh): `python precede_ocr.py YOUR_PDF_DIR --output-csv output/test_shutdown.csv`
2. Expected: "Resuming from checkpoint: X files already processed" message
3. Processing continues from where it left off (not from the beginning)
4. Let it complete normally or Ctrl+C again

**Test 4: Double Ctrl+C force-quit (D-03, D-04)**
1. Run: `python precede_ocr.py YOUR_PDF_DIR --output-csv output/test_forcequit.csv --fresh`
2. Wait for progress bar to show
3. Press Ctrl+C TWICE rapidly
4. Expected:
   - First press: "Ctrl+C received. Finishing in-flight files..."
   - Second press: "Force-quit! In-flight files may not be saved. Checkpoint has all completed files."
   - Process exits immediately

**Test 5: No zombie processes (SHUT-03)**
1. After any of the above tests, open Task Manager (Ctrl+Shift+Esc)
2. Check "Details" tab, sort by Name
3. Verify: No orphaned `python.exe` processes remain from the batch job
4. (There should be at most your terminal's Python process, not worker processes)

Wait for user to type "approved" or describe failures.
